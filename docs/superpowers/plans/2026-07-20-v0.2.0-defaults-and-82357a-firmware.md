# ugpibd v0.2.0 — Defaults + 82357A Firmware Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.2.0: safer/localhost-first defaults, a unified `--default-address`, opt-in Prologix, a quieter default log format, and in-process firmware loading for the Agilent 82357A.

**Architecture:** Two independent workstreams. (1) CLI/startup changes in `src/main.rs` plus a small threading of the default address through `src/server.rs`/`src/prologix.rs`. (2) Generalize the existing EZ-USB firmware uploader so the CPU-reset register is per-model, and bundle the 82357A image. No protocol or wire-format changes.

**Tech Stack:** Rust 2021, tokio (current-thread), clap (derive), tracing / tracing-subscriber, nusb.

## Global Constraints

- Rust edition 2021, `rust-version = "1.75"`.
- License header on every new `.rs` file: `// SPDX-License-Identifier: GPL-3.0-or-later`.
- Do not add third-party dependencies (this release adds none).
- The 82357A firmware is redistributed unmodified from `fmhess/linux_gpib_firmware` under the same terms as the existing 82357B blob; the `firmware/LICENSE` note must cover it.
- `cargo fmt` clean and `cargo clippy` warning-free before each commit.
- Every task ends green: `cargo test` passes.

```rust
/// Upload `firmware` (Intel HEX text) to an EZ-USB device in pre-init state.
/// Holds the 8051 in reset, writes all HEX records, then releases reset.
///
/// `cpucs_addr` is the CPU control/status register that resets the 8051:
/// `0xE600` on the FX2 (82357B), `0x7F92` on the first-gen EZ-USB / AN2131
/// (82357A). Everything else is identical across the two parts.
pub async fn upload_firmware(
    device: &nusb::Device,
    firmware: &[u8],
    cpucs_addr: u16,
) -> Result<()> {
    let hex_text = std::str::from_utf8(firmware).context("firmware blob is not valid UTF-8")?;
    let records = parse_hex(hex_text).context("failed to parse firmware HEX")?;

    tracing::info!("holding 8051 in reset");
    anchor_load(device, cpucs_addr, &[0x01])
        .await
        .context("failed to hold 8051 in reset")?;

    tracing::info!(records = records.len(), "writing firmware records");
    for record in &records {
        anchor_load(device, record.address, &record.data)
            .await
            .with_context(|| {
                format!("failed to write firmware record at {:#06x}", record.address)
            })?;
    }

    tracing::info!("releasing 8051 from reset");
    anchor_load(device, cpucs_addr, &[0x00])
        .await
        .context("failed to release 8051 from reset")?;

    Ok(())
}
```
