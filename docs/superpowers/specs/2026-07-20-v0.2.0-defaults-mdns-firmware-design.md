# ugpibd v0.2.0 — safer defaults, LXI mDNS discovery, 82357A firmware

Status: approved design (pending spec review)
Date: 2026-07-20

## Summary

Three groups of changes, released together as `0.2.0`:

1. **Defaults / CLI cleanup** — bind to localhost, unify the default-address
   arg, make Prologix opt-in, quiet the default log format.
2. **LXI-style mDNS discovery** (`--mdns`) — query `*IDN?` and advertise the
   attached instrument as HiSLIP + LXI DNS-SD services, designed to avoid the
   duplicate-`.local`-hostname conflict.
3. **Agilent 82357A firmware loading** — bundle the A's firmware image and
   parameterize the EZ-USB reset register so a cold 82357A can be brought up
   in-process, just like the 82357B.

Nothing here changes the on-bus protocol or the front-end wire protocols; the
work is startup configuration, a new optional discovery module, and a small
generalization of the firmware uploader.

---

## Part 1 — Defaults / CLI cleanup

### 1.1 Bind to localhost by default

`--bind` default changes `0.0.0.0` → `127.0.0.1`. Both listeners (HiSLIP and,
when enabled, Prologix) inherit it as today. Users who want remote access pass
`--bind 0.0.0.0` (or a specific interface address) explicitly.

### 1.2 Unify `--default-address`

Rename `--hislip-default-pad` → `--default-address`, **default `0`**, and use it
for both front-ends:

- HiSLIP: the fallback primary address when a client's subaddress does not
  encode one (was the sole consumer; its default moves 14 → 0).
- Prologix: seeds `PrologixState`'s initial `addr` (already 0, unchanged).

Consequences to document: a HiSLIP client using the bare `hislip0` subaddress
now targets pad 0 unless `--default-address` is set. mDNS users will normally
set `--default-address` (or `--mdns-address`) to their instrument's real pad.

`src/main.rs` is the only code consumer; `README.md` line ~72 is the only doc
reference.

### 1.3 Prologix opt-in

New flag `--enable-prologix` (bool, default `false`). HiSLIP stays enabled by
default (its `--hislip-port` default is unchanged). Implementation mirrors the
existing optional-HiSLIP pattern in `main.rs`:

- Build the Prologix `TcpListener` only when `--enable-prologix` is set;
  otherwise its branch of the `tokio::select!` is `std::future::pending()`.
- If **neither** front-end is enabled (`--hislip-port 0` and no
  `--enable-prologix`), exit with a clear error rather than idling silently.

### 1.4 Quieter default log format

Default level stays `ugpibd=info` (still reports backend/adapter discovery,
listeners bound, client connect/disconnect, and warnings/errors). The
`tracing_subscriber` formatter changes to reduce per-line noise:

- `.with_target(false)` — drop the `ugpibd::backend::agilent_82357::usb:` prefix.
- A compact time format (`.without_time()` or a short `%H:%M:%S` timer) so lines
  read like `INFO using backend agilent-82357b` instead of a full RFC3339 stamp
  plus module path.

`-v`/`-vv` and the `RUST_LOG` override are unchanged. This is formatter-only; no
log call sites move between levels.

### 1.5 Version

`Cargo.toml` `version = "0.1.0"` → `"0.2.0"`; update `Cargo.lock`. Minor bump:
additive features plus a couple of intentional default/CLI changes, pre-1.0.

## Part 2 — LXI-style mDNS discovery

### 2.1 Behavior

New flags:

- `--mdns` (bool, default `false`) — enable discovery advertisement.
- `--mdns-address <pad>` (optional u8) — instrument pad to identify; defaults to
  `--default-address` when omitted.

On startup, after the backend opens and **only if `--mdns` is set**:

1. Query `*IDN?` at the mDNS address (write `*IDN?\n`, read the response) through
   the shared backend.
2. Parse the response into `InstrumentId { manufacturer, model, serial,
   firmware }`.
3. Register two DNS-SD services and hold the daemon handle for the process
   lifetime:
   - `_hislip._tcp.local.` → the HiSLIP port. The connectable VISA endpoint.
   - `_lxi._tcp.local.` → identity beacon. Carries the identifying TXT records.

### 2.2 TXT records

Both services carry `txtvers=1`. `_lxi._tcp` additionally carries whatever of
these `*IDN?` yielded: `Manufacturer`, `Model`, `SerialNumber`,
`FirmwareVersion`. Missing fields are omitted rather than sent empty.

### 2.3 Crate choice — `mdns-sd`

Use the pure-Rust `mdns-sd` crate. Rationale:

- **No native/system build deps.** Consistent with the project's userspace,
  minimal-dependency stance (the alternative, `zeroconf`, links `libavahi-client`
  on Linux and needs avahi-daemon running).
- **Interoperates with the system responders.** `mdns-sd` co-binds UDP 5353 with
  `SO_REUSEADDR`/`SO_REUSEPORT` and joins the multicast group; its multicast
  advertisements are seen by macOS mDNSResponder, Linux avahi, and tools like
  `dns-sd -B` / `avahi-browse` and VISA discovery. It interoperates with the
  system responder rather than registering through it.

Adds a small `gethostname` (or equivalent) dependency for the fallback hostname.

### 2.4 Avoiding the duplicate-`.local`-hostname conflict (hard requirement)

The classic failure — a second responder claiming the machine's own
`myhost.local` while mDNSResponder/avahi already owns it, forcing a rename to
`myhost-2.local` and leaving stale records — must not happen. Mitigations, all
required:

1. **Register a dedicated, distinct hostname.** Never reuse the system's primary
   `.local` name. Use `ugpibd-<serial>.local.` where `<serial>` is the
   sanitized instrument serial from `*IDN?` (unique per instrument), falling back
   to `ugpibd-<system-hostname>.local.` when no serial is available. Sanitize to
   DNS-label-safe characters and length.
2. **Unique service instance name** derived from the same instrument identity
   (e.g. `"<Manufacturer> <Model>"`, or the hostname stem when identity is
   unavailable), so instances don't collide either.
3. **Graceful goodbye on shutdown.** On SIGINT/SIGTERM, `unregister()` the
   services (send DNS-SD goodbye packets) and give the daemon a brief moment to
   flush before exit, so a restart does not collide with lingering records.

### 2.5 Edge cases

- **`*IDN?` fails** (nothing at that address, timeout): log a warning and still
  advertise the HiSLIP endpoint, using a generic instance name and hostname
  (`ugpibd-<system-hostname>`) with no identity TXT. The endpoint is still valid
  and discoverable.
- **Loopback bind + `--mdns`:** log a prominent warning that the advertised
  endpoint is unreachable from other hosts and suggest `--bind 0.0.0.0`. Still
  advertise (locally discoverable is legitimate for testing).
- **HiSLIP disabled + `--mdns`:** error out — there is no HiSLIP endpoint to
  advertise.
- Advertised addresses follow the bind: `enable_addr_auto()` when bound to
  `0.0.0.0`, otherwise the specific bound address (loopback included).

### 2.6 `_lxi._tcp` SRV port — known simplification

ugpibd serves no HTTP, so `_lxi._tcp`'s SRV target has no natural port. It points
at the HiSLIP port and functions as an identity/TXT beacon only. Documented as a
deliberate simplification; tools that HTTP-GET the `_lxi` target will find no web
server, but discovery/identification still works.

### 2.7 Module structure

New `src/discovery.rs`, exported from `src/lib.rs`. Kept small and testable:

- `pub struct InstrumentId { manufacturer, model, serial, firmware }` — each
  field `Option<String>`.
- `pub fn parse_idn(resp: &[u8]) -> InstrumentId` — lenient: split on `,`, trim
  whitespace and trailing terminators; tolerate fewer than four fields.
- `pub struct DiscoveryConfig { hislip_port, bind_addr, hostname_stem, ... }`.
- `pub fn build_services(cfg, id) -> Vec<ServiceInfo>` — **pure**; constructs the
  two `ServiceInfo`s (types, instance name, hostname, port, TXT). This is the
  unit-tested core.
- `pub async fn query_idn(ctrl, pad) -> Result<InstrumentId>`.
- `pub async fn advertise(cfg, id) -> Result<ServiceDaemon>` — registers the
  built services and returns the live handle.

`main.rs` owns lifecycle: query → build → advertise → hold handle → unregister on
shutdown.

## Part 3 — Agilent 82357A firmware loading

### 3.1 Background

The 82357A is a first-generation Cypress EZ-USB (AN2131 / "FX") part. Its 8051
reset register (CPUCS) is at **`0x7F92`**, versus the 82357B's FX2 **`0xE600`**.
Confirmed against the upstream loader (`fmhess/linux_gpib_firmware`,
`agilent_82357a/firmware.c`) and the upstream README: the A is loaded by fxload
**without** `-t fx2` and **without** a second-stage loader, i.e. the exact same
`ANCHOR_LOAD_INTERNAL` (vendor request `0xA0`) sequence we already use for the B
— only the CPUCS reset address differs. So no external-RAM / two-stage path is
needed.

### 3.2 Changes

1. **Bundle `firmware/82357a_fw.hex`** from the same upstream project and the
   same redistribution terms as the existing B image. Extend `firmware/LICENSE`
   to cover the new file (it is for the 82357A; `measat_releaseX1.8.hex` is for
   the 82357B).
2. **Parameterize the reset register.** Add `cpucs_addr: u16` to
   `agilent_82357::Model`: `0xE600` for `MODEL_82357B`, `0x7F92` for
   `MODEL_82357A`. Thread it into `firmware::upload_firmware(...)` (replacing the
   hardcoded `CPUCS_ADDR`). The `0xA0` request and record-writing loop are
   unchanged. Rename the FX2-specific symbols to neutral names where they now
   cover both parts (e.g. keep `ANCHOR_LOAD_INTERNAL`; the `fx2_control` helper
   and `FX2_FIRMWARE_82357B` naming get generalized/renamed).
3. **Set `MODEL_82357A.firmware = Some(...)`** pointing at the bundled A image.
4. The existing double-upload retry loop in `initialize_device` is a B-specific
   quirk; for the A it harmlessly succeeds on the first attempt (renumeration to
   `0x0107`), so no branching is needed.
5. Update `README.md` (the adapter table and the 82357A note) and the module doc
   comments in `agilent_82357/mod.rs` / `firmware.rs`: the A is no longer
   "firmware not bundled." It remains **hardware-untested**; keep that caveat.

## Testing

- **Discovery (`tests/discovery.rs`):**
  - `parse_idn`: well-formed 4-field, trailing `\n`/terminator, extra whitespace,
    fewer than four fields, empty input.
  - `build_services`: asserts the two service types (`_hislip._tcp`, `_lxi._tcp`),
    the HiSLIP port, TXT keys/values, and that the hostname is a dedicated
    `ugpibd-*` name (never the bare system hostname).
  - Live multicast registration is not CI-testable (network/multicast); this is
    documented and out of automated scope.
- **Firmware:**
  - Parse the bundled `82357a_fw.hex` via `parse_hex` — validates the blob loads
    and is non-empty.
  - Guard `MODEL_82357A.firmware.is_some()` and the per-model CPUCS addresses
    (`A == 0x7F92`, `B == 0xE600`).
- **Existing suites** (`parse_hex`, prologix, hislip client/integration, scpi)
  must still pass. Full run: `cargo test`, plus `cargo clippy` and `cargo fmt`.

## Out of scope

- Raw SCPI (`_scpi-raw._tcp`, port 5025), VXI-11, and an HTTP/LXI web UI — the
  daemon serves none of these, so they are not advertised.
- Registering through the OS mDNS daemon (avahi/Bonjour) — see 2.3.
- Actual on-hardware validation of the 82357A path (no hardware available).
